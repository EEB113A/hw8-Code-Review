# Code Review 配分
每項最高4分，最低0分\
請各位同學直接複製以下的五個評分項目和審查意見，並貼到留言板上進行評分：

效能：\
模組化：\
程式碼的編排：\
有意義的命名：\
註解：\
審查意見：

```diff
+ 範例
效能：3
模組化：4

有意義的命名：3
註解：0
審查意見：我認為還有更好的寫法可以改善效能，程式碼看起來比較雜亂不意理解，命名易懂但可以更精確，但沒有加上註解使其他人不易讀懂。
- 注意！評分項目的冒號後面，只要打上一個數字就好，不要加任何其他字元！
```
